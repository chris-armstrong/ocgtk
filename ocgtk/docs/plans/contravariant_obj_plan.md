# Plan: Implement Contravariant Gobject.obj for Safe Layer 1 & Layer 2 Upcasting

## Executive Summary

Current ocgtk uses **covariant** `type +'a obj` in `gobject.ml`. This plan investigates whether changing to **contravariant** `type -'a obj` enables safe upcasting from derived to base types (e.g., `Button.t` → `Widget.t`), eliminating the need for `Obj.magic` in generated code.

**Key Insight**: With polymorphic variant types like `[`button | `widget | `initially_unowned] Gobject.obj`, the subtyping relationship is:
- `button` variant type ⊆ `widget` variant type (more specific)
- With proper variance, this allows safe upcasting

## Implementation Strategy

### Phase 0: Verification (TDD First)

**Goal**: Verify that contravariant `-'a obj` actually enables upcasting before modifying generators.

**Unit 0.1: Create Variance Test Suite**

Create `ocgtk/test/test_variance.ml`:

```ocaml
(* Test contravariant Gobject.obj upcasting *)

(* With contravariant -'a obj, we expect: *)
(* - Derived types can be upcast to base types via :> *)

let test_upcast_button_to_widget () =
  (* Button.t has polymorphic variant [`button | `widget | `initially_unowned] *)
  (* Widget.t has polymorphic variant [`widget | `initially_unowned] *)
  (* Button variant ⊆ Widget variant, so Button.t should upcast to Widget.t *)
  let btn = Button.new_ () in
  (* This should compile without Obj.magic *)
  let _ = (btn : Button.t :> Widget.t) in
  ()

let test_upcast_widget_to_initially_unowned () =
  let w = Widget.new_ () in
  let _ = (w : Widget.t :> Initially_unowned.t) in
  ()

let test_nested_upcast () =
  let btn = Button.new_ () in
  (* Direct upcast to base's base *)
  let _ = (btn : Button.t :> Initially_unowned.t) in
  ()
```

**Verification Criteria**:
- [ ] All upcast tests compile and pass with `-'a obj`
- [ ] Downcast tests still fail (type safety preserved)
- [ ] `dune build && dune runtest` succeeds

**Decision Gate**: If tests fail with contravariant `-'a obj`, stop and investigate. Do NOT proceed to code generation changes.
